# Cloudflare Worker: 网页元数据提取服务

> 面向新手工程师的详细说明

## 📋 项目概述

这个Cloudflare Worker的作用是**自动从URL中提取网页信息**（如标题、描述、图片），并存储到数据库中。

**类比：** 想象你在分享一个链接给朋友，这个程序就是自动帮你提取"这个链接是什么标题"、"它的描述是什么"、"配的什么图片"这些信息。

---

## 🔧 技术栈

- **Cloudflare Workers**: 云端函数，无需管理服务器
- **Cheerio**: HTML解析库（用于从网页提取信息）
- **D1数据库**: Cloudflare的SQLite数据库
- **JavaScript**: 编程语言

---

## 🚀 工作流程

```
用户发送请求 (包含URL)
    ↓
检查URL是否在数据库中
    ↓
如果有缓存 → 返回缓存数据
    ↓
如果没有 → 访问网页并提取信息
    ↓
保存到数据库
    ↓
返回提取的信息
```

---

## 📥 输入参数

这个Worker接收**POST请求**，包含以下数据：

| 参数名 | 类型 | 是否必需 | 说明 |
|--------|------|--------|------|
| `url` | 字符串 | ✅ | 要处理的网页链接 |
| `title` | 字符串 | ❌ | 前端已知的标题（若已知，跳过爬取） |
| `description` | 字符串 | ❌ | 前端已知的描述 |
| `imageUrl` | 字符串 | ❌ | 前端已知的图片URL |
| `contentType` | 字符串 | ❌ | 网页内容类型 |

**例子：**
```javascript
// 最简单的请求
{
  "url": "https://example.com"
}

// 前端已有信息的请求（跳过爬取）
{
  "url": "https://example.com",
  "title": "示例网站",
  "description": "这是一个示例",
  "imageUrl": "https://example.com/pic.jpg"
}
```

## 💡 核心逻辑详解

### 1️⃣ 快速返回：检查缓存

```javascript
const { results } = await stmt.all();  // 查询数据库
if (results && results.length > 0) {
  return Response.json({...results[0]});  // 有缓存就直接返回
}
```

**好处：** 避免重复访问同一个网站，节省时间和带宽。

---

### 2️⃣ 快速返回：某些网站跳过爬取

```javascript
const siteKeywords = ["facebook", "instagram", "amazon", ...];
for (let keyword of siteKeywords) {
  if (urlStructure.hostname.indexOf(keyword) > -1) {
    return Response.json({ title: "", desc: "", imageUrl: "" });
  }
}
```

**原因：** 这些网站可能有反爬虫措施或不需要提取信息，直接返回空值。

---

### 3️⃣ 特殊处理：Twitter/X

```javascript
if (urlStructure.hostname.indexOf("x.com") > -1 || 
    urlStructure.hostname.indexOf("twitter") > -1) {
  url = url.replace(urlStructure.hostname, "api.vxtwitter.com");
  isTwitter = true;  // 标记为Twitter
}
```

**说明：** 
- Twitter被转换到 `vxtwitter.com` API（这是一个第三方接口）
- 后面会用JSON格式解析结果，而不是HTML

---

### 4️⃣ 特殊处理：PTT论坛

```javascript
if (urlStructure.hostname.indexOf("ptt") > -1) {
  headers.cookie = "over18=1";  // 添加成人内容Cookie
}
```

**说明：** PTT是台湾知名论坛，默认需要年龄验证Cookie。

---

### 5️⃣ 主要工作：网页爬取

#### a) 获取网页内容

```javascript
let responseFrom = await fetch(url, {
  method: "GET",
  headers,  // 包含User-Agent等信息
  redirect: "follow"  // 跟随重定向
});
```

**headers说明：**
- `User-Agent`: 告诉服务器"我是浏览器"，避免被识别为爬虫
- `Accept-Language`: 告诉服务器使用繁体中文
- `cookie`: 某些网站需要

---

#### b) 处理编码格式

```javascript
let charset = "utf-8";  // 默认编码
if (contentType.indexOf("charset=") > -1) {
  charset = contentType.substring(...);  // 提取指定编码
}
```

**为什么重要：** 有些网站用GBK或Shift JIS编码，如果用错编码会显示乱码。

---

#### c) 提取HTML中的信息

```javascript
const soup = cheerio.load(html);  // 加载HTML

// 优先查找 <title> 标签
if (soup("title").length > 0)
  title = soup("title").text();
// 否则查找 og:title 元标签
else if (soup('meta[property="og:title"]').length > 0)
  title = soup('meta[property="og:title"]').attr("content");
```

**小知识 - 什么是元标签（Meta Tag）？**

```html
<!-- 传统标题 -->
<title>这是网页标题</title>

<!-- 社交媒体用的元标签（Open Graph） -->
<meta property="og:title" content="社交媒体显示的标题">
<meta property="og:description" content="社交媒体显示的描述">
<meta property="og:image" content="https://...图片URL">
```

---

#### d) 不同网站的图片提取策略

```javascript
if (urlStructure.hostname.indexOf("ptt") > -1) {
  // PTT: 从 <div class="richcontent"> 找图片
  imageUrl = soup("div.richcontent>img").attr("src");
} 
else if (urlStructure.hostname.indexOf("amazon") > -1) {
  // 亚马逊: 从特定ID的元素找
  imageUrl = soup("#landingImage").attr("src");
}
```

**原因：** 每个网站的HTML结构不同，需要对症下药。

---

### 6️⃣ 数据库操作：保存信息

```javascript
await DATABASE.prepare(`
  INSERT INTO urls VALUES (?, ?, ?, ?, ?)
  ON CONFLICT(url) DO UPDATE SET 
  title = excluded.title,
  desc = excluded.desc,
  imageUrl = excluded.imageUrl,
  contentType = excluded.contentType
`).bind(url, title, desc, imageUrl, contentType).run();
```

**这是什么？** **UPSERT操作**
- **INSERT**: 如果URL不存在，插入新记录
- **ON CONFLICT DO UPDATE**: 如果URL已存在，更新记录

**好处：** 一条SQL语句搞定"新增或更新"。

## 🐛 错误处理

```javascript
try {
  // 主要逻辑
} catch (e) {
  console.log(e);
  return Response.json({ "error": "发生错误\n" + e.toString() });
}
```

**常见错误：**
1. ❌ 没有提供URL → "No url"
2. ❌ 网站不可访问 → fetch错误
3. ❌ HTML解析失败 → cheerio错误
4. ❌ 数据库连接失败 → D1错误

## 🔄 优化机制

### ✅ 缓存机制
- 相同URL的请求直接返回缓存，无需重新爬取

### ✅ 前端优化
- 如果前端已知title/desc，可以直接发送，跳过爬取流程
- 减少服务器压力

### ✅ 网站黑名单
- Facebook、Instagram等网站直接返回空值
- 避免浪费资源在无法爬取的网站

## 📊 数据库表结构

```sql
CREATE TABLE urls (
  url TEXT PRIMARY KEY,      -- 网址（唯一标识）
  title TEXT,                -- 网页标题
  desc TEXT,                 -- 网页描述
  imageUrl TEXT,             -- 图片URL
  contentType TEXT           -- 内容类型
);
```

## 🎯 关键知识点

| 概念 | 说明 | 为什么重要 |
|------|------|---------|
| **Cheerio** | HTML解析库 | 用来从网页源码中提取信息 |
| **Meta标签** | 网页元信息 | 社交媒体和分享卡片都依赖这些 |
| **User-Agent** | 请求头 | 某些网站会根据这个判断是否爬虫 |
| **Charset编码** | 文本编码 | 错误的编码会导致乱码 |
| **UPSERT** | 插入或更新 | 高效的数据库操作 |
| **缓存** | 数据库查询 | 避免重复工作，提升性能 |

## ❓ 常见问题

### Q: 为什么要检查缓存？
A: 相同的URL可能被多个用户分享，缓存可以避免重复爬取同一个网站。

### Q: 为什么某些网站返回空值？
A: 有些网站（如Facebook）有反爬虫措施或不适合提取信息，直接返回空值节省资源。

### Q: User-Agent有什么用？
A: 模拟浏览器身份。有些网站会拒绝爬虫的请求，使用User-Agent可以"伪装"成浏览器。

### Q: 为什么需要特殊处理Twitter？
A: Twitter/X的HTML结构复杂，使用vxtwitter.com的API接口更简单高效。

### Q: 数据库中的数据多久清理一次？
A: 取决于业务需求，可以手动维护或设置定期清理任务。
